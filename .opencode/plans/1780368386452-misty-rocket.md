## Visión General

Implementar el parser Rust que extrae el programa semanal desde el PDF de la guía de actividades (mwb), Y la interfaz React completa para importar con selección de semanas. Al terminar, el usuario puede abrir un PDF, ver todas las semanas parseadas (~8-9), elegir cuáles importar, revisar/editar inline, y guardarlas en la base de datos.

### Descubrimiento clave del análisis del PDF real

El PDF (`refs/mwb_S_202603.pdf`) usa **caracteres de control ASCII** como marcadores estructurales:

| Hex | Carácter | Significado |
|-----|----------|-------------|
| `0x02` (STX) | `` | Separador en week header / marca sección Tesoros |
| `0x03` (ETX) | `` | Marca sección Seamos Mejores Maestros |
| `0x04` (EOT) | `` | Marca sección Nuestra Vida Cristiana |
| `0x05` (ENQ) | `` | Precede "Canción N" |
| `0x0C` (FF) | `` | Salto de página |

Formato de encabezado de semana: `"2 - 8 D E MAR ZO  I SA ÍAS 4 1 , 4 2"`
Formato de parte: `"1. Título (X mins.)"`
Formato de canción: `" Canción 89 y oración"`

> **Nota**: `lopdf.extract_text()` puede o no preservar estos caracteres de control. El parser debe ser robusto ante ambas posibilidades. Si no se preservan, usaremos patrones de texto plano como fallback.

## Tarea 2.3: Extracción de texto y detección de semanas

**Archivo**: `src-tauri/src/mwb_parser/mod.rs`

**Qué hacer**:

### 2.3a — `extract_text_from_pdf(path)`
```rust
fn extract_text_from_pdf(path: &str) -> Result<String, String> {
    let doc = Document::load(path).map_err(|e| format!("Error al abrir PDF: {}", e))?;
    let pages: Vec<u32> = doc.get_pages().keys().cloned().collect();
    let text = doc.extract_text(&pages)
        .map_err(|e| format!("Error al extraer texto: {}", e))?;
    Ok(text)
}
```

### 2.3b — `split_into_weeks(raw_text)`
La estrategia es de 2 pasos:

**Paso 1 — Limpiar texto**:
- Reemplazar `\r\n` → `\n`
- Collapsar múltiples saltos de línea (>2) en máximo 2
- Normalizar caracteres de control: `\x05` → `[SONG]`, `\x02` → ` `, `\x03` → ` `, `\x04` → ` `
- Eliminar espacios entre letras de meses: `"MAR ZO"` → `"MARZO"`, `"AB R I L"` → `"ABRIL"`

**Paso 2 — Detectar encabezados de semana**:
Regex para fechas: `r"(\d{1,2})\s*[-–]\s*(\d{1,2})\s+(?:D\s*E\s+)?(\w+)"` (captura día inicio, día fin, mes)

Cada vez que se encuentra un encabezado de semana, se inicia un nuevo bloque. El texto entre dos encabezados pertenece a la primera semana.

### 2.3c — Detectar semanas especiales
Antes de parsear cada semana, buscar en el bloque de texto:
- `"ASAMBLEA"` o `"asamblea"` → `TipoEspecial::Asamblea`
- `"CONMEMORACIÓN"` o `"Conmemoración"` → `TipoEspecial::Conmemoracion`
- `"VISITA DEL SUPERINTENDENTE"` → `TipoEspecial::VisitaSuperintendente`

Las semanas de Asamblea y Conmemoración se marcan como `SinReunion` (sin partes que asignar). Se importan pero solo ocupan espacio en el calendario.

### Manejo de errores
- Archivo no encontrado → `Err("No se encontró el archivo: {path}")`
- PDF corrupto → `Err("El archivo no es un PDF válido")`
- Sin texto extraíble → `Err("No se pudo extraer texto del PDF")`

**Validación**: Con `refs/mwb_S_202603.pdf`, la función `extract_text_from_pdf` retorna texto no vacío. `split_into_weeks` detecta 8-9 bloques. ~1.5-2h.

## Tarea 2.4: Parseo de cada semana

**Archivo**: `src-tauri/src/mwb_parser/mod.rs`

**Qué hacer**:

### 2.4a — `parse_week(week_text) -> Result<ParsedWeek, String>`

**Extraer libro bíblico**:
Del encabezado de semana, extraer la parte después del separador. Limpiar espacios extra entre letras y números.
- Ejemplo: `"2 - 8 D E MAR ZO  I SA ÍAS 4 1 , 4 2"` → `"Isaías 41, 42"`

**Parsear fechas**:
Del encabezado, extraer día_inicio, día_fin, mes. Convertir a ISO 8601. El año se toma del nombre del archivo (mwb_S_**2026**03.pdf) o se asume el año actual.

**Detectar canciones**:
Buscar `[SONG]` o `Canción`/`Cántico` seguido de número:
- `r"(?:\[SONG\]|Canción|Cántico)\s*(\d+)"`
- Primera ocurrencia → `cancion_apertura`
- La que aparece justo antes o dentro de "NUESTRA VIDA CRISTIANA" → `cancion_intermedia`
- Última ocurrencia → `cancion_cierre`

**Detectar secciones**:
Buscar encabezados de sección (insensibles a mayúsculas y tolerantes a espacios):
- `"TESOROS DE LA BIBLIA"` → cambia contexto a `tesoros`
- `"SEAMOS MEJORES MAESTROS"` → cambia contexto a `mejores_maestros`
- `"NUESTRA VIDA CRISTIANA"` → cambia contexto a `vida_cristiana`

**Detectar partes**:
Regex: `r"(\d+)\.\s+(.+?)\s+\((\d+)\s*(?:min|mins)\.?\)"`

Para cada coincidencia:
1. Extraer `numero_orden`, `titulo`, `duracion_minutos`
2. Asignar `seccion` según el contexto actual (última sección detectada)
3. Clasificar `tipo_asignacion` con `classify_part_type(titulo, seccion)`

Partes especiales a detectar (no tienen número):
- `"Palabras de introducción"` + `(1 min.)` → `Introduccion`
- `"Palabras de conclusión"` + `(3 min.)` → `Conclusion`

### 2.4b — `classify_part_type(title, section) -> (String, bool, bool)`

Tabla de keywords en orden de prioridad (más específico primero):

| Keyword en título | TipoAsignacion | Sala Aux | Ayudante |
|---|---|---|---|
| `Busquemos perlas escondidas` | `busquemos_perlas` | No | No |
| `Lectura de la Biblia` | `lectura_biblia` | Sí | No |
| `Empiece conversaciones` | `empiece_conversaciones` | Sí | Sí |
| `Haga revisitas` | `haga_revisitas` | Sí | Sí |
| `Haga discípulos` | `haga_discipulos` | Sí | Sí |
| `Explique sus creencias` + contiene `Discurso` | `explique_creencias_discurso` | Sí | No |
| `Explique sus creencias` (sin Discurso) | `explique_creencias_escenificacion` | Sí | Sí |
| `Análisis con el auditorio` | `analisis_auditorio` | No | No |
| `Necesidades de la congregación` | `necesidades_congregacion` | No | No |
| `Estudio bíblico de la congregación` | `estudio_biblico` | No | No |
| `Discurso` en sección `mejores_maestros` | `discurso_estudiante` | Sí | No |
| `Discurso` en sección `tesoros` | `discurso_no_estudiante` | No | No |
| `Discurso` en sección `vida_cristiana` | `discurso_no_estudiante` | No | No |
| Cualquier otro título en `tesoros` (parte 1) | `discurso_no_estudiante` | No | No |
| Cualquier otro en `mejores_maestros` | `discurso_estudiante` | Sí | No |
| Cualquier otro en `vida_cristiana` | `discurso_no_estudiante` | No | No |

**Validación**: Para una semana del mwb real, >90% de partes clasificadas correctamente. ~2-3h.

```rust
#[tauri::command]
pub fn import_parsed_weeks(
    db: State<'_, Database>,
    parsed_weeks: Vec<mwb_parser::ParsedWeek>,
) -> Result<Vec<Semana>, String> {
    let conn = db.conn.lock().map_err(|e| e.to_string())?;
    let mut results = Vec::new();

    conn.execute("BEGIN", []).map_err(|e| e.to_string())?;

    let result = (|| -> Result<Vec<Semana>, String> {
        for pw in &parsed_weeks {
            conn.execute(
                "INSERT INTO semanas (fecha_inicio, fecha_fin, libro_biblico,
                 cancion_apertura, cancion_intermedia, cancion_cierre, tipo_especial)
                 VALUES (?1, ?2, ?3, ?4, ?5, ?6, ?7)",
                rusqlite::params![
                    pw.fecha_inicio,
                    pw.fecha_fin,
                    pw.libro_biblico,
                    pw.cancion_apertura,
                    pw.cancion_intermedia,
                    pw.cancion_cierre,
                    match pw.tipo_especial {
                        TipoEspecial::Normal => "normal",
                        TipoEspecial::Asamblea => "asamblea",
                        TipoEspecial::Conmemoracion => "conmemoracion",
                        TipoEspecial::VisitaSuperintendente => "visita_superintendente",
                        TipoEspecial::SinReunion => "asamblea", // usar "asamblea" para sin reunion
                    },
                ],
            ).map_err(|e| e.to_string())?;

            let semana_id = conn.last_insert_rowid();

            for pp in &pw.partes {
                conn.execute(
                    "INSERT INTO partes (semana_id, numero_orden, seccion,
                     tipo_asignacion, titulo, duracion_minutos,
                     requiere_sala_auxiliar, requiere_ayudante)
                     VALUES (?1, ?2, ?3, ?4, ?5, ?6, ?7, ?8)",
                    rusqlite::params![
                        semana_id,
                        pp.numero_orden,
                        pp.seccion,
                        pp.tipo_asignacion,
                        pp.titulo,
                        pp.duracion_minutos,
                        pp.requiere_sala_auxiliar as i32,
                        pp.requiere_ayudante as i32,
                    ],
                ).map_err(|e| e.to_string())?;
            }

            results.push(Semana {
                id: semana_id,
                fecha_inicio: pw.fecha_inicio.clone(),
                fecha_fin: pw.fecha_fin.clone(),
                libro_biblico: Some(pw.libro_biblico.clone()),
                cancion_apertura: Some(pw.cancion_apertura as i32),
                cancion_intermedia: Some(pw.cancion_intermedia as i32),
                cancion_cierre: Some(pw.cancion_cierre as i32),
                tipo_especial: /* ... */,
                presidente_id: None,
                consejero_sala_id: None,
                orador_oracion_apertura_id: None,
                orador_oracion_intermedia_id: None,
                orador_oracion_cierre_id: None,
            });
        }
        Ok(results)
    })();

    match result {
        Ok(r) => { conn.execute("COMMIT", []).map_err(|e| e.to_string())?; Ok(r) }
        Err(e) => { conn.execute("ROLLBACK", []).ok(); Err(e) }
    }
}
```

## Notas Técnicas Importantes

1. **Caracteres de control**: El texto extraído por `lopdf.extract_text()` puede no incluir los caracteres `\x02`-`\x05`. El parser debe funcionar con y sin ellos. Priorizar patrones de texto plano (`"Canción"`, `"TESOROS DE LA BIBLIA"`) sobre caracteres de control.

2. **Mes con espacios**: `"MAR ZO"`, `"AB R I L"` — el texto del PDF inserta espacios aleatorios entre letras. Usar regex flexible o tabla de meses conocidos.

3. **Formato de fecha ISO 8601**: El parser debe producir `"2026-03-02"`. El año se infiere del nombre del archivo o se hardcodea como el año actual.

4. **Partes sin número**: "Palabras de introducción" y "Palabras de conclusión" no tienen número de parte. Deben detectarse por texto.

5. **`blocking_pick_file` vs `pick_file`**: Usamos la versión blocking porque estamos en un comando Tauri sync. El diálogo bloquea el thread pero es aceptable para este caso de uso.
